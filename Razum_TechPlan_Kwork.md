## ЭТАП 1: MVP (2–3 месяца, ~250 тыс. ₽)

### Блок А: Форк и ребрендинг Gonka (50–80 тыс. ₽)

**Кворк А1: Клонировать и запустить Gonka локально**
- Что: склонировать github.com/gonka-ai/gonka, установить все зависимости, запустить локальную тестовую сеть на Ubuntu
- Требования к исполнителю: Go, Docker, Cosmos SDK
- Результат: работающая локальная нода + инструкция как запустить
- Бюджет: 10–20 тыс. ₽
- Где искать: Kwork → "блокчейн разработка", "Cosmos SDK"

**Кворк А2: Ребрендинг кода**
- Что: заменить "Gonka" → "Razum", "GNK" → "RZM" во всём коде, конфигах, документации
- Требования: знание Go, grep/sed, git
- Результат: чистый форк с названием Razum, компилируется и работает
- Бюджет: 5–10 тыс. ₽

**Кворк А3: Новый genesis-блок**
- Что: создать свой genesis.json с нуля (не копировать Gonka), настроить начальное распределение токенов RZM
- Требования: Cosmos SDK
- Результат: своя независимая сеть Razum
- Бюджет: 10–15 тыс. ₽

**Кворк А4: Настройка ML-ноды**
- Что: запустить ML-ноду с моделью Llama 3 на арендованном GPU-сервере (Vast.ai ~$0.5/час)
- Требования: Python, PyTorch, vLLM, Docker, CUDA
- Результат: нода принимает запросы и выдаёт ответы
- Бюджет: 15–25 тыс. ₽

**Кворк А5: Связать блокчейн + ML-ноду**
- Что: убедиться что задачи из блокчейна попадают на ML-ноду, результаты записываются в блокчейн
- Требования: Go, Python, gRPC
- Результат: полный цикл: отправка задачи → вычисление → запись результата
- Бюджет: 10–15 тыс. ₽

### Блок Б: Веб-интерфейс (40–70 тыс. ₽)

**Кворк Б1: Сайт-лендинг**
- Что: сверстать красивый лендинг (у нас уже есть index.html, нужно доработать)
- Требования: HTML, CSS, JS
- Результат: готовый лендинг на Netlify
- Бюджет: 3–5 тыс. ₽ (или бесплатно — уже готов)

**Кворк Б2: Веб-приложение (чат с AI)**
- Что: Next.js приложение с чатом, стримингом ответов, выбором модели, историей
- Требования: React, Next.js, TypeScript, WebSocket/SSE
- Результат: рабочий чат, как ChatGPT, подключённый к нашей ноде
- Бюджет: 20–40 тыс. ₽

**Кворк Б3: Авторизация пользователей**
- Что: регистрация, вход, профиль, хранение истории чатов
- Требования: Next.js, PostgreSQL или Firebase
- Результат: пользователь регистрируется, его чаты сохраняются
- Бюджет: 10–15 тыс. ₽

**Кворк Б4: Интеграция Stripe**
- Что: подключить оплату подписки картой (Free/Basic $20/Pro $50)
- Требования: Stripe API, Next.js
- Результат: пользователь оплачивает подписку, получает кредиты
- Бюджет: 10–15 тыс. ₽

## КАК ЗАКАЗЫВАТЬ НА KWORK — ПРАВИЛА

### Поиск исполнителей
- Ищи по ключевым словам: "блокчейн", "Cosmos SDK", "React Native", "Next.js", "Stripe"
- Смотри рейтинг: 4.5+ звёзд, 50+ выполненных заказов
- Читай отзывы — особенно негативные
- Проси показать портфолио до заказа

### Как составлять задание
- Чёткое ТЗ = хороший результат. Размытое ТЗ = проблемы
- Пиши: что на входе, что на выходе, какие технологии
- Прикладывай файлы (наш white paper, техплан)
- Укажи сроки
- Разбивай большие задачи на маленькие

### Пример хорошего ТЗ для кворка

```
ЗАДАЧА: Настроить форк блокчейна Gonka на локальном сервере

ЧТО НУЖНО СДЕЛАТЬ:
1. Склонировать https://github.com/gonka-ai/gonka
2. Установить все зависимости (Go, Docker, и т.д.)
3. Запустить локальную тестовую сеть из 1 ноды
4. Убедиться что нода работает и принимает транзакции
5. Написать инструкцию как это повторить

СЕРВЕР: Ubuntu 22.04 (предоставлю доступ по SSH)
РЕЗУЛЬТАТ: работающая нода + документация
СРОК: 5–7 дней
```

### Пример плохого ТЗ

```
Сделайте мне блокчейн для AI. Чтобы было как биткоин, 
но для нейронок. И мобильное приложение. И токен.
```

Так не надо — исполнитель не поймёт что делать, результат будет плохой.

### Защита от рисков
- Всегда используй "Безопасную сделку" на Kwork (деньги замораживаются до приёмки)
- Первый кворк с новым исполнителем — маленькая задача (5–10К)
- Если хорошо сделал — давай следующую, покрупнее
- Сохраняй все исходники, доступы, пароли у себя
