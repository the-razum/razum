# Razum — Полный технический план разработки

## Всё, что нужно построить, в каком порядке, на чём, кто нужен, сколько стоит

---

## ОБЩАЯ КАРТИНА

Проект состоит из 7 больших блоков:

```
1. Блокчейн (ядро сети)
2. GPU-сеть (ноды, которые считают AI-задачи)
3. Система верификации (проверка честности нод)
4. Координатор (кто какую задачу получает)
5. Платёжная система (подписка + токен)
6. Веб-приложение (сайт, как ChatGPT)
7. Мобильные приложения (iOS + Android)
```

Каждый блок разбит на этапы. Всего 4 больших этапа, каждый — 3–6 месяцев.

## ЭТАП 1: MVP (месяцы 1–4)

**Цель:** работающий прототип, который можно показать инвесторам и первым пользователям.

### 1.1 Простой блокчейн

**Что делаем:**
- Берём готовый фреймворк Cosmos SDK (язык Go) — не пишем блокчейн с нуля
- Это как взять WordPress вместо того, чтобы писать CMS с нуля
- Настраиваем свою цепочку с токеном RZM
- Базовые смарт-контракты: регистрация ноды, отправка задачи, начисление награды

**Технологии:**
- Cosmos SDK (Go) — фреймворк для создания блокчейнов
- Tendermint — консенсус-движок (идёт в комплекте с Cosmos)
- CometBFT — новая версия Tendermint

**Почему Cosmos, а не Ethereum/Solana:**
- Cosmos позволяет создать свой отдельный блокчейн, а не смарт-контракт на чужом
- Полный контроль над правилами сети
- Быстрый (6 секунд на блок vs 12 у Ethereum)
- Межсетевое взаимодействие через IBC (потом пригодится)

**Кто нужен:** 1 блокчейн-разработчик (Go, Cosmos SDK)
**Сроки:** 6–8 недель
**Стоимость:** $15 000–25 000 (фрилансер) или со-основатель за долю

### 1.2 GPU-нода (клиент для майнеров)

**Что делаем:**
- Программа, которую владелец GPU скачивает и запускает
- Она автоматически: подключается к сети, скачивает нужную модель, принимает задачи, выполняет их, отправляет результат
- На старте поддерживаем 3–5 моделей: Llama 3 (чат), Stable Diffusion (картинки), Whisper (голос в текст), Mistral (чат), BGE (embeddings)

**Технологии:**
- Python — основной язык
- PyTorch — запуск моделей
- vLLM — быстрый инференс для LLM (та самая скорость 0.5–1 сек)
- CUDA — работа с GPU NVIDIA
- Docker — упаковка всего в контейнер (чтобы "скачал и запустил")

**Оптимизации скорости (сразу закладываем):**
- vLLM с PagedAttention — ускоряет инференс в 2–4 раза
- Модели в формате GPTQ/AWQ — сжатие без потери качества
- Предзагрузка моделей в GPU-память (горячие ноды)
- gRPC вместо HTTP для общения между нодами (быстрее в 5–10 раз)

**Кто нужен:** 1 ML-инженер (Python, PyTorch, CUDA)
**Сроки:** 4–6 недель
**Стоимость:** $10 000–20 000 или со-основатель

### 1.3 Простой координатор задач

**Что делаем:**
- Сервер, который принимает задачу от пользователя
- Находит свободную ноду с нужной моделью
- Отправляет задачу, получает результат, возвращает пользователю
- На этом этапе — централизованный (один сервер), децентрализуем позже

**Технологии:**
- Go или Rust — быстрый бэкенд
- Redis — очередь задач и кэширование
- PostgreSQL — база данных (ноды, задачи, пользователи)
- gRPC — связь с нодами
- WebSocket — стриминг ответа пользователю (посимвольно, как ChatGPT)

**Кто нужен:** 1 бэкенд-разработчик (Go/Rust)
**Сроки:** 4–6 недель
**Стоимость:** $10 000–20 000

### 1.4 Веб-интерфейс (сайт)

**Что делаем:**
- Страница чата — как ChatGPT, пишешь вопрос, получаешь ответ
- Стриминг ответа (текст появляется посимвольно)
- Регистрация / вход
- Выбор модели (Llama, Mistral, Stable Diffusion)
- Страница "мой аккаунт" (баланс, история, подписка)
- Лендинг — объяснение проекта

**Технологии:**
- React + Next.js — фронтенд
- TypeScript — язык
- Tailwind CSS — стилизация
- Vercel — хостинг (бесплатно на старте)

**Кто нужен:** 1 фронтенд-разработчик
**Сроки:** 4–6 недель
**Стоимость:** $8 000–15 000

### 1.5 Простая оплата

**Что делаем:**
- Stripe для приёма карт (подписка $20/$50/$200)
- При оплате на баланс пользователя зачисляются "кредиты"
- Кредиты списываются за каждый запрос
- Токен RZM на этом этапе не нужен — только внутренние кредиты

**Почему без токена на старте:**
- Stripe подключается за 1 день, крипто-платежи — за месяцы
- Обычные люди привыкли платить картой
- Токен добавим на Этапе 2, когда будет рабочий продукт

**Технологии:**
- Stripe API — подписки и платежи
- Webhook — автоматическое начисление кредитов

**Кто нужен:** тот же бэкенд-разработчик
**Сроки:** 1–2 недели (Stripe очень простой)
**Стоимость:** входит в стоимость бэкенда

## ЭТАП 2: Продукт (месяцы 5–9)

**Цель:** продукт, которым реально пользуются люди, мобильные приложения, токен.

### 2.1 Мобильные приложения (iOS + Android)

**Два подхода:**

**Подход А: React Native (один код → два приложения)**
- Один разработчик пишет код, получается и iOS, и Android
- Быстрее и дешевле
- Чуть медленнее нативных приложений (но для чата — незаметно)
- Используют: Instagram, Discord, Coinbase

**Подход Б: Нативные приложения (отдельно iOS и Android)**
- Swift для iOS, Kotlin для Android
- Быстрее работают, лучше интеграция с системой
- Нужно 2 разработчика вместо 1
- Дороже в 2 раза

**Рекомендация:** React Native на старте, нативные — потом, если нужна производительность.

**Функции приложения:**
- Чат с AI (стриминг, как в ChatGPT)
- Голосовой ввод (Whisper через сеть)
- Генерация картинок
- Управление подпиской
- Push-уведомления
- Тёмная тема

**Технологии:**
- React Native + Expo — кроссплатформенная разработка
- TypeScript — язык
- React Navigation — навигация
- Expo Notifications — пуши

**Публикация:**
- App Store: нужен Apple Developer Account ($99/год), ревью 1–3 дня
- Google Play: нужен Google Developer Account ($25 разово), ревью 1–7 дней
- Важно: Apple берёт 30% с подписок через App Store (но можно направить на сайт)

**Кто нужен:** 1 React Native разработчик
**Сроки:** 6–8 недель
**Стоимость:** $15 000–25 000

### 2.2 Запуск токена RZM

**Что делаем:**
- Токен RZM на блокчейне Cosmos
- Кошелёк в веб-приложении и мобильном
- Обмен: кредиты подписки ↔ RZM (внутри системы)
- Стейкинг для нод
- Награды за вычисления в RZM

**Важно — юридическая сторона:**
- Консультация с крипто-юристом (обязательно!)
- Регистрация фонда (Швейцария, Сингапур или Каймановы острова)
- KYC/AML для крупных транзакций
- Стоимость юриста: $20 000–50 000

**Кто нужен:** блокчейн-разработчик (тот же) + юрист
**Сроки:** 4–6 недель (код) + 4–8 недель (юридическое)
**Стоимость:** $30 000–60 000 (включая юриста)

### 2.3 Система верификации v1

**Что делаем (все решения из нашего обсуждения):**

**Для инференса — выборочное дублирование:**
- 15% задач случайно отправляются на 2 дополнительные ноды
- Результаты сравниваются автоматически
- Несовпадение → слэшинг (штраф 5% стейка)

**Для генерации — скрытые тесты:**
- 5% задач — тестовые с известным ответом
- Нода не знает, какая задача тестовая
- Провал теста → снижение репутации

**Для тренировки — чекпоинты:**
- Чекпоинт каждые 500 шагов
- Валидатор проверяет случайный сегмент (50 шагов)
- Порог отклонения — калиброванный по типу GPU

**Технологии:**
- Python — логика верификации
- NumPy — сравнение тензоров с допуском
- Отдельный микросервис верификации

**Кто нужен:** ML-инженер (тот же) + 1 дополнительный
**Сроки:** 6–8 недель
**Стоимость:** $15 000–25 000

### 2.4 Оптимизации скорости

**Всё, что мы обсуждали:**

**Географические кластеры:**
- Определяем регион пользователя по IP
- Задача идёт на ближайшие ноды
- Первые хабы: Европа, Россия/СНГ, США, Азия
- Технология: GeoIP + умная маршрутизация в координаторе

**Горячие ноды:**
- Популярные модели (Llama, Mistral) всегда в памяти GPU
- Повышенная награда за "дежурство" (+20% к обычной ставке)
- Мониторинг: какие модели запрашивают чаще, автоматический прогрев

**Стриминг:**
- WebSocket соединение между пользователем и координатором
- Server-Sent Events (SSE) как fallback
- Первый токен через 0.3–1 сек (vLLM + горячая нода + близкий регион)

**Кэширование:**
- Redis для кэша ответов на популярные запросы
- Semantic cache — похожие вопросы получают кэшированный ответ
- Embedding-модель определяет "похожесть" запросов
- Экономия: до 30% GPU-ресурсов

**Кто нужен:** бэкенд-разработчик + ML-инженер
**Сроки:** 4–6 недель
**Стоимость:** входит в зарплаты команды

## ЭТАП 3: Масштабирование (месяцы 10–15)

### 3.1 Распределённая тренировка и файнтюнинг
- Пользователь загружает свои данные → модель обучается на распределённых GPU
- Технологии: DeepSpeed, PyTorch Distributed
- Шифрование данных через TEE (Intel SGX / AMD SEV)

### 3.2 AI Marketplace
- Каталог моделей (как App Store, но для AI-моделей)
- Создатели моделей публикуют, получают % от использования
- Рейтинги, отзывы, категории

### 3.3 API для разработчиков
- REST API + SDK (Python, JavaScript, Go)
- Документация (как у OpenAI)
- Биллинг по использованию (pay-per-token)
- Совместимость с OpenAI API формат (легко мигрировать)

### 3.4 Децентрализация координатора
- Убираем центральный сервер
- Координация через смарт-контракты на блокчейне
- P2P-сеть (libp2p) для прямого общения нод

### 3.5 DAO
- Голосование держателей токенов
- Предложения по изменениям протокола
- Управление казной
- Модель: Compound/Uniswap governance

**Команда на этом этапе:** 8–12 человек
**Стоимость:** $150 000–300 000

## ЭТАП 4: Полный продукт (месяцы 16–24)

### 4.1 Enterprise-решения
- Выделенные ноды для корпоративных клиентов
- SLA (гарантия скорости и доступности)
- On-premise вариант (сеть внутри компании)
- Интеграция с корпоративными системами

### 4.2 ZK-Proofs верификация v2
- Zero-Knowledge доказательства для верификации
- Убирает необходимость в дублировании
- Технологии: RISC Zero, SP1

### 4.3 Кросс-чейн
- Мосты к Ethereum, Solana, Polygon
- Оплата в ETH, SOL, USDC
- IBC (Inter-Blockchain Communication) через Cosmos

### 4.4 Экосистема
- Грантовая программа ($1M+ для разработчиков)
- Хакатоны
- Партнёрства с университетами
- Open-source инструменты

**Команда:** 15–25 человек
**Стоимость:** $500K–1M+ (к этому моменту должно быть финансирование)

## ПРИОРИТЕТЫ: ЧТО ДЕЛАТЬ ПЕРВЫМ

Строгий порядок — каждый шаг зависит от предыдущего:

```
Неделя 1:     Настроить Cosmos SDK, запустить тестовую сеть на ноутбуке
Неделя 2–3:   Написать GPU-ноду (Python + vLLM + одна модель Llama)
Неделя 4–5:   Связать ноду с блокчейном (задача → вычисление → результат)
Неделя 6–7:   Координатор (принять задачу, найти ноду, вернуть ответ)
Неделя 8–9:   Веб-интерфейс (чат со стримингом)
Неделя 10:    Stripe подписка
Неделя 11–12: Тестирование на 3–5 нодах, исправление багов
Неделя 13–14: Запуск для первых 10–50 пользователей (закрытая бета)
Неделя 15–16: Мобильные приложения (React Native)
Неделя 17–18: Токен RZM, стейкинг
Неделя 19–20: Верификация v1, репутация нод
```
