# Contributing to Razum AI

Спасибо за интерес. Razum AI — open source проект, любая помощь приветствуется.

## Что можно делать

### Без писания кода
- Сообщать баги (issues)
- Предлагать фичи
- Улучшать документацию (PR в .md файлы)
- Переводы на другие языки
- Делиться отзывами в Telegram @razum_ai

### Код
- Чинить open issues с лейблом `good first issue`
- Улучшать существующие фичи
- Писать тесты
- Оптимизация (latency, размер бандла)
- UI улучшения

### Если ты майнер
- Запустить ноду и держать её в онлайне — самый прямой вклад
- Сообщать о странном поведении сети
- Документировать свой опыт настройки на разных платформах

## Процесс PR

1. Форкни репо: github.com/the-razum/razum
2. Создай ветку: `feat/моя-фича` или `fix/мой-баг`
3. Делай маленькие коммиты с понятными сообщениями
4. Проверь что smoke-test проходит: `bash web-app/scripts/smoke-test.sh`
5. Открой PR с описанием: что изменено и зачем
6. Получи review (обычно за 1-3 дня)
7. После approve — мержим

## Соглашение о коде

- TypeScript для web-app, Solidity для контрактов, Go для chain-side
- ESLint без новых warnings
- Не использовать `any` без причины
- Все API-routes должны иметь rate limit
- Все mutating endpoints должны быть CSRF-защищены (через middleware) либо явно exempt с обоснованием
- Без логов с personal data (email, IP в logs только в production debug режиме)

## Архитектура — основные правила

- Координатор (web-app) не делает inference сам. Только маршрутизирует
- Майнеры подписывают каждый ответ ECDSA
- Все длинноживущие state — в БД или на чейне, не в process memory
- Не доверять входным данным от майнеров без подписи
- Не доверять входным данным от юзеров без zod-валидации

## Кодекс поведения

- Уважай других участников
- Конструктивная критика, без личных нападок
- Английский или русский — оба ОК
- Не публикуй чужой код без атрибуции
- Не пытайся продавить свой токен/проект в наш чат
