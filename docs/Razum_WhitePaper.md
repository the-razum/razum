# Razum: Децентрализованная сеть AI-вычислений

## White Paper v0.1 — Апрель 2026

---

## 1. Проблема

Сегодня доступ к AI-вычислениям контролируется горсткой корпораций: NVIDIA, AWS, Google Cloud, Microsoft Azure. Это создаёт три критические проблемы:

**Дефицит GPU.** Обучение одной frontier-модели требует тысячи GPU на месяцы. Очереди на H100/B200 измеряются кварталами. Малые команды и исследователи отрезаны от вычислительных мощностей.

**Непрозрачное ценообразование.** Облачные провайдеры устанавливают цены без рыночной конкуренции. Стоимость GPU-часа может отличаться в 3–5 раз между провайдерами без видимых причин.

**Централизация контроля.** Кто контролирует вычисления — контролирует, какие модели создаются. Vendor lock-in, закрытые API, возможность отключить доступ в любой момент.

При этом миллионы GPU по всему миру простаивают: игровые ПК, майнинговые фермы после перехода Ethereum на PoS, университетские кластеры вне рабочих часов.

---

## 2. Решение: Razum

Razum — это децентрализованная сеть, где:

- **Вместо бесполезного хеширования** (Proof-of-Work в Bitcoin) вычислительные мощности выполняют реальные AI-задачи: инференс моделей, тренировку, файнтюнинг.
- **Участники получают токены (RZM)** за выполненные задачи, пропорционально вкладу.
- **Любой пользователь** может отправить задачу в сеть и оплатить её в RZM.
- **Результаты верифицируются** через механизм Proof-of-Useful-Compute (PoUC).

### 3.2 Слой 1: Блокчейн и вычислительные ноды

**Консенсус: Proof-of-Useful-Compute (PoUC)**

В отличие от Bitcoin (PoW) и Ethereum (PoS), Razum использует гибридный подход:

- Ноды выполняют реальные AI-задачи (инференс, тренировка, файнтюнинг).
- Каждая выполненная задача = вклад в безопасность сети.
- Для участия нода должна застейкать минимальный объём RZM (защита от атак).
- Вес голоса ноды = f(стейк × вычислительный вклад × репутация).

**Типы нод:**

| Тип | Требования | Роль | Награда |
|-----|-----------|------|---------|
| Compute Node | GPU (RTX 3060+) + стейк 1000 RZM | Выполнение AI-задач | 70% от комиссии задачи |
| Validator Node | Стейк 10000 RZM | Верификация результатов | 15% от комиссии |
| Storage Node | 1 TB+ SSD + стейк 500 RZM | Хранение моделей/данных | 10% от комиссии |
| Relay Node | Стабильный интернет | Маршрутизация задач | 5% от комиссии |

### 3.3 Слой 2: Верификация (Proof-of-Useful-Compute)

Главный вызов: как проверить, что нода правильно выполнила AI-задачу?

**Три механизма верификации:**

**a) Redundant Execution (для инференса)**
Одна задача отправляется 3 случайным нодам. Результаты сравниваются. Если 2 из 3 совпадают (с допуском ε для floating-point), результат принимается. Несогласная нода теряет часть стейка (слэшинг).

**b) Gradient Checkpointing (для тренировки)**
При тренировке модели ноды отправляют контрольные точки (чекпоинты) каждые N шагов. Валидаторы случайно проверяют сегменты, прогоняя те же данные. Расхождение выше порога → слэшинг.

**c) Challenge-Response (для сложных задач)**
Любой валидатор может бросить "челлендж" — запросить повторное вычисление фрагмента. Нода должна воспроизвести результат за отведённое время. Если не может → слэшинг + потеря задачи.

### 3.4 Слой 3: Координация

**Планировщик задач (Task Scheduler):**
- Пользователь отправляет задачу: тип (инференс/тренировка), модель, данные, бюджет.
- Планировщик определяет требуемые ресурсы (VRAM, compute units).
- Матчинг-алгоритм подбирает оптимальные ноды по: мощности GPU, латентности, репутации, цене.
- Задача разбивается на подзадачи при необходимости (distributed training).

**Система репутации:**
- Каждая нода имеет Reputation Score (0–1000).
- Успешное выполнение задач → +reputation.
- Провалы, таймауты, несовпадения → −reputation.
- Низкая репутация → меньше задач → меньше дохода.
- Высокая репутация → приоритетные и дорогие задачи.

### 3.5 Слой 4: Приложения

**AI Marketplace:**
- Каталог моделей (open-source и кастомные).
- Единый API для инференса любой модели.
- Простая интеграция через SDK (Python, JS, Rust).

**Примеры использования:**
- Стартап запускает LLM-инференс без контракта с AWS.
- Исследователь файнтюнит модель на распределённом кластере.
- Компания хостит приватную модель с шифрованием данных.
- Разработчик создаёт AI-агента, оплачивая вычисления в RZM.

## 4. Токеномика

### 4.1 Токен RZM

**Общее предложение:** 1 000 000 000 RZM (1 миллиард)
**Эмиссия:** Дефляционная модель, halvings каждые 4 года (аналогия с Bitcoin)

### 4.2 Распределение

| Категория | Доля | Количество | Вестинг |
|----------|------|-----------|---------|
| Вычислительные награды | 50% | 500M | Выпуск через 20+ лет |
| Команда и советники | 15% | 150M | 4 года, cliff 1 год |
| Экосистемный фонд | 15% | 150M | Разблокировка по этапам |
| Ранние инвесторы | 10% | 100M | 3 года, cliff 6 мес |
| Публичная продажа | 5% | 50M | Без блокировки |
| Ликвидность | 5% | 50M | Частично заблокировано |

### 4.3 Экономическая модель

**Как RZM обретает ценность:**

1. **Utility-спрос.** Пользователи покупают RZM, чтобы оплатить AI-вычисления. Больше пользователей → больше спрос на RZM.
2. **Стейкинг.** Ноды блокируют RZM для участия в сети. Больше нод → больше заблокировано → меньше в обращении.
3. **Fee Burn.** 20% от каждой комиссии сжигается навсегда. Дефляционное давление.
4. **Halving.** Награда за блок уменьшается вдвое каждые 4 года. Уменьшение притока новых токенов.

**Ценообразование задач:**
- Цены устанавливаются рынком (аукцион между нодами).
- Ориентир: на 30–60% дешевле облачных провайдеров.
- Минимальная комиссия сети: 2% от стоимости задачи.

## 7. Риски и митигация

| Риск | Вероятность | Митигация |
|------|-----------|-----------|
| Низкое качество вычислений | Высокая | PoUC + система репутации + слэшинг |
| Атака Sybil (фейковые ноды) | Средняя | Обязательный стейк + proof-of-hardware |
| Латентность выше облака | Высокая | Фокус на batch-задачах, не real-time |
| Регуляторные риски | Средняя | Юрисдикция фонда в Швейцарии/Сингапуре |
| Конкуренция от Render/Akash | Высокая | Уникальный фокус на тренировку + верификацию |
| Недостаток ликвидности | Средняя | Стратегические партнёрства с DEX |
