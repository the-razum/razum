# Razum AI — План запуска продукта

**Дата:** 9 апреля 2026
**Статус:** Рабочий прототип → нужно довести до production

---

## Текущее состояние: честная оценка

Razum AI сейчас — **работающий proof-of-concept**. Чат работает end-to-end: пользователь пишет → VPS ставит задачу в очередь → майнер на Mac Mini подхватывает → Ollama генерирует → ответ стримится обратно. Это ~2200 строк кода ядра (координатор + майнер).

Но до продукта, за который платят — дистанция серьёзная. Ниже — что именно делать.

## 1. ПРОДУКТ — что должен уметь пользователь

### Что работает сейчас

- Регистрация/логин (email + пароль, PBKDF2 100K итераций)
- Чат с 2 моделями: DeepSeek R1 14B, Mistral 7B
- Стриминг ответов в реальном времени
- Веб-поиск (Brave/DuckDuckGo) для актуальной информации
- 4 тарифа: Free (30/день), Start (500/день, 490р), Basic (2000/день, 990р), Pro (безлимит, 1990р)
- Страница майнера с калькулятором заработка
- Анонимный доступ (10 запросов/день без регистрации)

### Что НЕ работает / не дописано

| Фича | Статус | Приоритет |
|------|--------|-----------|
| Оплата через Робокассу | Вебхук есть, верификация подписи не реализована | **БЛОКЕР** |
| Срок действия тарифа | `planExpiresAt` в БД есть, логика продления НЕ написана | **БЛОКЕР** |
| Подтверждение email | Отсутствует полностью | Высокий |
| Сброс пароля | Отсутствует полностью | Высокий |
| Личный кабинет | Страница-заглушка, нет истории, нет статистики | Высокий |
| Генерация картинок | Отсутствует | Средний |
| Выбор из 5+ моделей | Только 2 модели | Средний |
| История чатов | Нет серверного хранения, теряется при перезагрузке | Высокий |
| API-ключи | Отсутствуют, нет программного доступа | Средний |
| Удаление аккаунта | Отсутствует | Средний |

### Что нужно, чтобы пользователь не ушёл на ChatGPT

1. **Скорость ответа < 3 сек до первого токена** — сейчас зависит от одного Mac Mini. Нужно минимум 3-5 GPU-нод.
2. **Надёжность 99.5%** — сейчас очередь в памяти, рестарт = потеря задач. Нужен Redis или SQLite-очередь.
3. **Качество моделей** — DeepSeek R1 14B хорош для русского, но нужны Llama 3.1 70B, Qwen 2.5 72B для конкуренции.
4. **История чатов** — без этого сервис неюзабелен для повседневного использования.
5. **Рабочая оплата** — пока Робокасса не проверяет подпись, платные тарифы — фикция.

## 2. ТЕХНОЛОГИЯ — что не дописано, где заглушки

### Архитектура (текущая)

```
Пользователь → Next.js (VPS, PM2) → globalThis TaskQueue → Miner (Mac Mini) → Ollama → обратно
```

### Критические проблемы

**2.1. Очередь задач — в памяти (globalThis)**
- Рестарт PM2 = потеря всех задач в очереди
- Не масштабируется на несколько процессов/серверов
- **Решение:** Перенести очередь в SQLite (таблица `tasks` с PENDING/ASSIGNED/COMPLETED) или Redis. SQLite проще и уже подключен.
- **Сложность:** 1-2 дня
- **Приоритет:** БЛОКЕР

**2.2. Rate limiter — в памяти**
- Рестарт = сброс всех лимитов
- Код содержит TODO: "Replace with Redis for multi-instance"
- **Решение:** Хранить счётчики в SQLite (таблица `rate_limits`)
- **Сложность:** 0.5 дня
- **Приоритет:** Высокий

**2.3. Одна GPU-нода**
- Один Mac Mini M4 = один пользователь за раз, остальные ждут
- **Решение фаза 1:** Добавить 2-3 ноды (свои или волонтёры). Мейнер уже поддерживает мульти-нод маршрутизацию.
- **Решение фаза 2:** Docker-образ для майнера (Dockerfile есть, но entrypoint.sh НЕ НАПИСАН)
- **Сложность:** 2-3 дня (Docker) + время на поиск нод
- **Приоритет:** БЛОКЕР для > 10 пользователей

**2.4. Нет персистентной истории чатов**
- Чаты хранятся только в localStorage браузера
- **Решение:** Таблица `chats` + `messages` в SQLite, API `/api/chats`
- **Сложность:** 2-3 дня
- **Приоритет:** Высокий

**2.5. Вебхук оплаты**
- Роут `/api/payment/webhook` существует, но не верифицирует подпись Робокассы
- **Решение:** Реализовать MD5-проверку: `md5(OutSum:InvId:pass2)`, активировать тариф при совпадении
- **Сложность:** 0.5 дня
- **Приоритет:** БЛОКЕР

**2.6. Custom server — single process**
- `server.js` запускает Next.js в одном процессе (для globalThis). Это правильно для текущей архитектуры, но потолок ~50 одновременных запросов.
- **Решение при росте:** Redis/SQLite очередь → PM2 cluster mode → Nginx load balancing
- **Приоритет:** После 100+ DAU

### Где заглушки в коде

| Файл | Строка/модуль | Что заглушено |
|------|--------------|---------------|
| `db.ts` | `upgradePlan()` | Не устанавливает `planExpiresAt` |
| `payment/webhook/route.ts` | Весь файл | Нет верификации подписи |
| `account/page.tsx` | Весь файл | UI-заглушка без данных |
| `miner/Dockerfile` | entrypoint.sh | Файл отсутствует в репо |
| `taskQueue.ts` | In-memory | Нет персистентности |

## 3. БЛОКЧЕЙН И ТОКЕНОМИКА

### Что есть

- **RZMToken.sol** — полноценный ERC-20 контракт:
  - 1B RZM, 18 decimals
  - 1% комиссия трансферов (20% сжигается, 80% в казну)
  - `rewardMiner(address, amount, taskId)` — начисление наград

- **Токеномика:** 50% майнеры, 15% команда, 15% комьюнити, 10% инвесторы, 10% ликвидность

## 4. ИНФРАСТРУКТУРА

### Текущее состояние

- **VPS:** Timeweb, PM2 процесс "razum-web" порт 18003
- **Домен:** airazum.com (SSL через Nginx)
- **БД:** SQLite файл на VPS
- **Майнер:** Mac Mini M4 (192.168.0.107), Ollama локально
- **CI/CD:** Нет. Деплой вручную через SCP + PM2 restart
- **Мониторинг:** Только `/api/health` эндпоинт
- **Бэкапы:** Нет

### Что нужно сделать

| Задача | Решение | Стоимость | Приоритет |
|--------|---------|-----------|-----------|
| CI/CD | GitHub Actions: lint → test → build → deploy (SSH) | Бесплатно | Высокий |
| Бэкапы БД | Cron-скрипт: `sqlite3 .backup` → S3/Backblaze B2 | ~$1/мес | БЛОКЕР |
| Мониторинг | Uptime Kuma (self-hosted) или Better Stack free tier | Бесплатно | Высокий |
| Логи | PM2 logs + logrotate. Позже: Grafana Loki | Бесплатно | Средний |
| Второй VPS | Для Redis/бэкап ноды при росте | ~$10/мес | После 100 DAU |
| Docker registry | GitHub Container Registry (ghcr.io) для майнер-образа | Бесплатно | Высокий |
| Staging-сервер | Отдельный PM2 процесс на том же VPS, порт 18004 | Бесплатно | Средний |

### Минимальный CI/CD пайплайн

```yaml
# .github/workflows/deploy.yml
name: Deploy
on:
  push:
    branches: [main]
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm ci && npm run build  # в web-app/
      - run: npm test  # когда тесты появятся
      - name: Deploy via SSH
        uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.VPS_HOST }}
          username: ${{ secrets.VPS_USER }}
          key: ${{ secrets.VPS_KEY }}
          script: |
            cd /opt/razum-web && git pull && npm ci && npm run build && pm2 restart razum-web
```

## 5. БЕЗОПАСНОСТЬ

### Что уже хорошо

- Аутентификация: HMAC-SHA256 токены, timing-safe сравнение, httpOnly secure cookies
- Пароли: PBKDF2 100K итераций, индивидуальные salt (NIST SP 800-132)
- Rate limiting: 5 рег/час, 10 логинов/15 мин с экспоненциальным бэкоффом (30→480 сек)
- CSP, X-Frame-Options: DENY, X-Content-Type-Options: nosniff
- CORS: только для `/api/miners` и `/api/network`
- Валидация: email RFC 5322, пароль 8-128, сообщения 8000 символов макс, размер запроса 100KB
- Защита от enum: "email или пароль неверен" (не раскрывает существование аккаунта)

### Что нужно добавить

| Уязвимость | Риск | Решение | Приоритет |
|------------|------|---------|-----------|
| Rate limiter в памяти | Сброс при рестарте | SQLite хранение | Высокий |
| Нет бэкапов БД | Потеря всех данных | Автоматические бэкапы | БЛОКЕР |
| Мейнер API без подписи запросов | Спуфинг | HMAC подпись + timestamp | Средний |
| Нет rate limit по юзеру (только IP) | Один юзер может исчерпать ресурсы | Per-user лимиты | Высокий |
| SECRET захардкожен в dev mode | Утечка при неправильной конфигурации | Fail-fast если SECRET не задан | Высокий |
| SQLite без шифрования | Доступ к серверу = доступ к данным | SQLCipher или дисковое шифрование | Низкий |
| Нет логирования security-событий | Невозможно расследовать инциденты | Audit log таблица | Средний |
| Нет 2FA | Слабая защита аккаунтов | TOTP (Google Authenticator) | Низкий |
| Нет CAPTCHA | Бот-регистрации | hCaptcha (бесплатно) | Средний |

## 9. КОНКУРЕНТНОЕ ПРЕИМУЩЕСТВО

### Конкуренты и позиционирование

| Конкурент | Что делают | Слабость | Наше преимущество |
|-----------|-----------|----------|-------------------|
| **Gonka AI** | Decentralized AI + blockchain consensus | Сложный, медленный onboarding | Простота: зашёл → пишешь → работает |
| **Akash Network** | Decentralized cloud compute | Не заточен под AI, сложный UI | AI-first UX, no DevOps needed |
| **Bittensor** | AI subnet marketplace | Нужен TAO стейк ($$$), для разработчиков | Бесплатный tier для обычных пользователей |
| **ChatGPT** | Централизованный AI | Цензура, $20/мес, нет крипто | Без цензуры, бесплатный tier, заработок для нод |
| **Together AI** | API для моделей | Только API, нет chat UI | Полноценный чат + API |

### Что сделает Razum проектом №1

1. **"ChatGPT, но без цензуры и бесплатно"** — это hook. DeepSeek R1 отвечает на вопросы, которые ChatGPT отказывается. Это реальный спрос.

2. **Зарабатывай на своём GPU** — простейший onboarding для майнеров: `npx razum-miner` → автодетект моделей → начал зарабатывать. Без Docker, без настройки.

3. **Двойная экономика** — пользователи платят рублями/криптой за запросы → деньги распределяются между GPU-нодами → ноды мотивированы работать. Flywheel.

4. **Русскоязычный рынок** — ни один конкурент не имеет нормального русского UI и поддержки. Мы — первые.

5. **OpenAI-совместимый API** — razum-openai SDK уже есть (Go, Python, TypeScript). Drop-in replacement: поменяй `base_url` и работай.

### Что конкретно нужно сделать лучше Gonka

| Gonka | Razum сейчас | Что заимствовать |
|-------|-------------|-----------------|
| PoW 2.0 (100% ресурсов на AI) | Простой poll-based | Концепцию reputation-based валидации |
| Probabilistic verification (1-10%) | Нет верификации | Рандомизированная проверка результатов |
| BLS multi-sig | Нет | ПРОПУСТИТЬ (overengineered для < 200 нод) |
| Cosmos SDK chain | Централизованный координатор | ПРОПУСТИТЬ пока. Переход на chain — после product-market fit |
| StreamVesting | Нет | Добавить вестинг контракты перед токен-запуском |

### Что мы уже лучше Gonka

- **Работающий чат-интерфейс** — у Gonka нет consumer-facing продукта
- **Простота майнера** — один файл vs Docker + Cosmos node + GPU setup
- **Русский рынок** — Gonka полностью англоязычный
- **Модульность** — легко добавить новые модели, изменить UX
