# ЧАСТЬ 1: ЧТО ТАКОЕ NEURALCHAIN

## Идея в одном предложении

Razum — это децентрализованная сеть, где вместо бессмысленного хеширования (как в Bitcoin) вычислительные мощности GPU выполняют реальные AI-задачи (инференс, тренировка, файнтюнинг), а участники получают за это токены.

## Аналогия

Bitcoin: миллионы GPU считают бессмысленные хеши → никому не нужная работа → зато сеть защищена.
Razum: те же GPU выполняют реальные AI-задачи → полезная работа → и сеть защищена, и люди получают AI-сервис.

Для пользователя это выглядит как ChatGPT с платной подпиской (картой, без крипто-кошельков), но под капотом — блокчейн и распределённая GPU-сеть.

## Ключевое отличие от конкурентов

Существующие проекты (Render, Akash, Bittensor, Golem, IO.NET) закрывают отдельные части:
- Render — GPU-рендеринг, расширяется в AI
- Akash — децентрализованный облачный хостинг
- Bittensor — рынок AI-моделей
- Golem — аренда вычислительных ресурсов

Razum объединяет ВСЁ: инференс + тренировка + файнтюнинг + криптографическая верификация результатов (Proof-of-Useful-Compute с тремя методами). Плюс простая подписка картой для обычных пользователей.

## Типы нод

| Тип | Требования | Роль | Награда |
|-----|-----------|------|---------|
| Compute Node | GPU (RTX 3060+) + стейк 1000 RZM | Выполнение AI-задач | 70% от комиссии задачи |
| Validator Node | Стейк 10000 RZM | Верификация результатов | 15% от комиссии |
| Storage Node | 1 TB+ SSD + стейк 500 RZM | Хранение моделей/данных | 10% от комиссии |
| Relay Node | Стабильный интернет | Маршрутизация задач | 5% от комиссии |

## Поток задачи в сети

1. Пользователь отправляет запрос через сайт/приложение/API
2. Координатор за 0.1 сек подбирает ближайшую ноду с нужной моделью
3. Нода выполняет вычисление, стримит ответ посимвольно
4. Система верификации проверяет честность (2 из 3 совпадают)
5. Результат → пользователю, награда в RZM → ноде

# ЧАСТЬ 3: ВЕРИФИКАЦИЯ (Proof-of-Useful-Compute)

## Проблема

Как проверить, что удалённая нода реально выполнила AI-задачу, а не подсунула мусор?

## Решения по типам задач

### Для инференса — выборочное дублирование
- 15% задач случайно отправляются на 2 дополнительные ноды
- Результаты сравниваются автоматически
- Нода не знает, какая задача будет проверена (как камеры скорости)
- Несовпадение → слэшинг (штраф 5% стейка)
- Новая нода — проверяем 50% задач, опытная (10000+ задач) — 1%

### Для генерации — скрытые тесты
- 5% задач — тестовые с заранее известным правильным ответом
- Нода не знает, что это тест
- Тесты неотличимы от обычных задач
- Провал теста → снижение репутации + штраф

### Для тренировки — чекпоинты с ловушками
- Нода сохраняет состояние модели каждые 500 шагов
- Валидатор случайно выбирает чекпоинт, прогоняет 50 шагов сам
- Допустимый коридор отклонений (калибруется заранее по разным GPU)
- Дополнительно: проверка прогресса (loss должен уменьшаться, а не прыгать хаотично)
- Расхождение выше порога → слэшинг

### На будущее — Zero-Knowledge Proofs
- Математическое доказательство правильности вычислений без раскрытия данных
- Пока слишком медленно (100–1000x замедление)
- Через 3–5 лет может стать практичным
- Заложить как "модуль будущего" в архитектуру

## Итоговая стратегия верификации

Этап 1: выборочное дублирование + скрытые тесты + чекпоинты
Этап 2: добавить ZK-proofs по мере удешевления технологии

# ЧАСТЬ 4: ОПТИМИЗАЦИИ СКОРОСТИ

## Проблема

ChatGPT отвечает за 1–2 секунды (дата-центр). Децентрализованная сеть медленнее из-за сетевых задержек, поиска ноды и верификации. Без оптимизаций: 5–15 секунд.

## Решения (все будем использовать)

### 1. Географические кластеры
- Ноды группируются по регионам (Москва, Берлин, Токио...)
- Запрос пользователя идёт на ближайшие ноды
- Определяем регион по IP
- Задержка падает с 200 мс до 10–30 мс

### 2. Горячие ноды с предзагруженными моделями
- Популярные модели (Llama, Mistral) всегда в памяти GPU
- Не нужно каждый раз загружать 40 ГБ модели
- Горячие ноды получают +20% к награде за "дежурство"
- Мониторинг: какие модели запрашивают чаще, автоматический прогрев

### 3. Умная верификация вместо тотальной
- Новая нода → проверяем 50% задач
- Нода с хорошей репутацией (1000+) → 5%
- Нода с отличной репутацией (10000+) → 1%
- Одна нода вместо трёх = скорость как у централизованного сервиса

### 4. Стриминг ответа
- WebSocket соединение с пользователем
- Нода начинает генерировать → первые токены сразу летят
- Верификация в фоне, после показа ответа
- Первый токен через 0.3–1 сек с горячей ноды

### 5. Кэширование
- Redis для кэша ответов на популярные запросы
- Semantic cache — похожие вопросы получают кэшированный ответ
- Embedding-модель определяет "похожесть"
- Экономия до 30% GPU-ресурсов

## Реалистичные скорости после оптимизаций

- Без оптимизаций: 5–15 сек (неприемлемо)
- С кластерами + горячие ноды: 2–4 сек (терпимо)
- Со стримингом: первое слово через 0.5–1 сек (как ChatGPT)
- С кэшированием популярных запросов: мгновенно

Пользователь платит картой как за Netflix. Не знает про блокчейн и токены.

### Для крипто-участников — токен RZM

Владельцы GPU получают RZM за вычисления. Держатели RZM голосуют за изменения (DAO). Инвесторы покупают RZM, веря в рост проекта.

## Куда идут деньги с подписки

Пользователь заплатил $20:
- $12 (60%) → GPU-нодам за вычисления
- $2 (10%) → валидаторам за проверку
- $2 (10%) → казна проекта (разработка, серверы)
- $2 (10%) → сжигание токенов (дефляция)
- $2 (10%) → команда (операционный доход)

## Зачем нужен токен, если есть подписка

1. Мотивация нод — ранние майнеры получают токены, которые могут вырасти в цене
2. Децентрализация управления — DAO, голосование
3. Инвестиционный инструмент — финансирование без венчуров

# ЧАСТЬ 6: ТОКЕНОМИКА

## Токен RZM

- Общее предложение: 1 000 000 000 RZM
- Эмиссия: дефляционная, halving каждые 4 года

## Распределение

| Категория | Доля | Вестинг |
|----------|------|---------|
| Вычислительные награды | 50% | Выпуск через 20+ лет |
| Команда и советники | 15% | 4 года, cliff 1 год |
| Экосистемный фонд | 15% | По этапам |
| Ранние инвесторы | 10% | 3 года, cliff 6 мес |
| Публичная продажа | 5% | Без блокировки |
| Ликвидность | 5% | Частично заблокировано |

## Механизмы роста цены

1. Utility-спрос — оплата AI-задач только в RZM
2. Стейкинг — ноды блокируют RZM для участия
3. Fee Burn — 20% каждой комиссии сжигается навсегда
4. Halving — награда ÷2 каждые 4 года

# ЧАСТЬ 7: СЛАБЫЕ МЕСТА И РИСКИ

## 1. Проблема курицы и яйца
Нужны ноды → для этого нужны пользователи → для этого нужны ноды.
Решение: щедрые награды ранним майнерам из пула 50%.

## 2. Латентность
Децентрализованная сеть медленнее дата-центра.
Решение: географические кластеры + горячие ноды + стриминг (см. Часть 4).

## 3. Верификация — нерешённая проблема индустрии
Как проверить, что нода честна?
Решение: три метода PoUC + система репутации (см. Часть 3).

## 4. Конкуренты далеко впереди
Render с 2017, Bittensor с 2021.
Решение: уникальная ниша — инференс + тренировка + файнтюнинг + простая подписка.

## 5. Экономика может не сходиться
Нужно быть дешевле AWS, но при этом платить нодам.
Решение: фокус на batch-задачи + overhead ниже за счёт отсутствия дата-центров.

## 6. Регуляторные риски
Токен = возможно ценная бумага.
Решение: крипто-юрист с самого начала, фонд в Швейцарии/Сингапуре.

## 7. Безопасность данных
Данные пользователя на чужом GPU.
Решение: TEE (Trusted Execution Environments) на Pro-плане.

## 8. Нужна репутация и комьюнити
Без личного бренда тяжело привлечь людей.
Решение: контент (Telegram-канал, Twitter), выступления, white paper.

# ЧАСТЬ 15: ФОРК GONKA — СТРАТЕГИЯ

## Что такое Gonka

Gonka AI — проект братьев Даниила и Давида Либерманов. Децентрализованная инфраструктура для AI-вычислений. Запущен в августе 2025 года. Bitfury инвестировал $50M. Протокол open source, контроль передан сообществу. У Либерманов осталось 10% с локом на 4 года.

Gonka делает ровно то же, что мы проектировали: GPU-ноды выполняют AI-задачи вместо бессмысленного хеширования, участники получают токены GNK, разработчики получают дешёвый доступ к вычислениям.

## Решение: ФОРК + СВОИ ФИЧИ

Берём весь код Gonka как базу. Переименовываем. Добавляем то, чего у них НЕТ.

### Что есть у Gonka

- Рабочий блокчейн на Cosmos SDK
- ML-ноды с PyTorch/vLLM
- Proof-of-Compute консенсус
- OpenAI-совместимый API
- SDK для разработчиков (Python, TS, Go)
- Система эпох и наград
- Деплой-скрипты

### Чего НЕТ у Gonka (наши конкурентные преимущества)

1. **Подписка картой (Stripe)** — у Gonka только крипто-оплата. Обычный пользователь не может просто зайти и начать пользоваться. Мы добавим Stripe, планы $0/$20/$50/$200, автоконвертацию долларов в токены внутри системы.

2. **Чат-интерфейс как у ChatGPT** — у Gonka только API и CLI. Нет красивого сайта, где можно просто написать вопрос. Мы добавим веб-интерфейс со стримингом, выбором моделей, историей чатов.

3. **Мобильные приложения (iOS + Android)** — у Gonka нет. React Native, чат, генерация картинок, управление подпиской, push-уведомления.

4. **AI Marketplace** — каталог моделей, рейтинги, простая интеграция. Как App Store для AI-моделей.

5. **Фокус на СНГ рынок** — локализация, ноды в России/СНГ для низкой латентности, оплата в рублях/тенге, русскоязычное комьюнити.

6. **Географические кластеры** — маршрутизация на ближайшие ноды для скорости.

7. **Semantic кэширование** — кэш ответов на похожие вопросы для мгновенного ответа.

Система определяет страну по IP → показывает подходящие способы. Россия → ЮKassa. Другие → Stripe. Везде → крипто.

Цены для России: Free 0₽, Basic 1500₽/мес, Pro 3900₽/мес, Business 15000₽/мес.

Уровень 2 (после роста): вывод на карту — Россия через ЮKassa, мир через Stripe Connect, СНГ через Kaspi/Uzcard.
Уровень 3 (после листинга): продажа RZM на бирже (MEXC, Gate.io).

В приложении: Кошелёк → баланс → "Вывести" → выбор способа → реквизиты → деньги. Как банковское приложение.

## 17.3 Подключение майнеров — "3 клика"

# ЧАСТЬ 21: ДВА СЛОЯ — ОБЫЧНЫЙ БИЗНЕС + БЛОКЧЕЙН

## 21.1 Принцип "ресторан"

Razum = два независимых слоя.

"Зал" (снаружи) — обычный SaaS-сервис. Сайт, приложение, подписка, ЮKassa, ООО, расчётный счёт. Как Netflix, Яндекс.Плюс. Полностью законно.

"Кухня" (внутри) — децентрализованная GPU-сеть. Блокчейн, ноды, токен RZM, верификация. Как Bitcoin, но для AI. Никто не контролирует.

Пользователь видит только "зал". Майнер видит "кухню". Основатель управляет "залом", "кухня" работает автоматически.

## 21.2 Как работает оплата российской картой

1. Пользователь нажимает "Подписка 490 ₽"
2. Перекидывает на ЮKassa (как Wildberries/Ozon)
3. Вводит карту МИР или СБП
4. Деньги → расчётный счёт ООО "Разум" в российском банке
5. Сервер получает уведомление → начисляет кредиты

Никакого блокчейна в этом процессе нет. Обычный платёж.

## 21.3 Движение денег

```
ПОЛЬЗОВАТЕЛЬ → 490 ₽ картой → ЮKassa → р/с ООО
    ├── 60% → выкуп RZM у майнеров / оплата GPU
    ├── 15% → серверы, хостинг
    ├── 15% → развитие (Kwork, маркетинг)
    └── 10% → прибыль основателя

ПАРАЛЛЕЛЬНО:
МАЙНЕР → выполняет задачу → блокчейн начисляет RZM
    → обменник: RZM → USDT → рубли на карту
```

Два контура параллельно. Рубли — через ООО. Токены — через блокчейн.

## 21.4 Где блокчейн, а где нет

БЕЗ блокчейна: регистрация пользователя, оплата подписки, личный кабинет, история чатов, кредиты.

С блокчейном: учёт работы майнеров, начисление RZM, стейкинг, торговля токеном, DAO-голосование.

Оплата подписки и блокчейн — два отдельных контура. Пересекаются только когда деньги от подписок идут на выкуп RZM у майнеров.

## 21.5 Юридическая структура (Россия)

ООО на УСН 6%:
- Регистрация: через Тинькофф за 0 ₽, онлайн, 3–5 дней
- ОКВЭД: 62.01 (разработка ПО) + 63.11 (обработка данных)
- ЮKassa: заявка на yookassa.ru, нужен ИНН юрлица, подключение 1–3 дня
- Налог: 6% от выручки (УСН "доходы")

Законность:
- ООО + подписка на AI = полностью законно (как любой SaaS)
- Токен RZM: в РФ крипто не запрещена, но нельзя использовать как средство платежа внутри РФ
- Поэтому пользователь платит РУБЛЯМИ через ЮKassa, не токенами
- Токен — для майнеров и крипто-рынка
- Рекомендация: консультация крипто-юриста (5–15 тыс. ₽)

- Профиль — настройки, подписка, нода

- Ретеншн: мессенджер открывают 50 раз/день (AI-чат — 1 раз)

Razum = Signal (шифрование) + Telegram (мессенджер) + ChatGPT (AI) + Bitcoin (крипто)

## 25.1 Доступ к перепискам

Вариант А (Signal): полный E2EE, никто не может читать. Максимум приватности, но конфликт с законом Яровой.
Вариант Б (Telegram): обычные чаты — шифрование на сервере (можем читать, но не читаем). Секретные чаты — полный E2EE. Формально соответствует законам.

Решение: Вариант Б на старте. Проходим модерацию, не конфликтуем с законами. Пользователям даём выбор.

## 25.2 Модерация App Store / Google Play

Мессенджер с шифрованием — ✅ пропустят (Signal, Telegram — в магазинах).
AI-чат — ✅ пропустят (ChatGPT, Claude — в магазинах).
Крипто-кошелёк — ⚠️ сложнее:
- Можно: показывать баланс, историю, внутренний обмен
- Нельзя: покупка крипто за фиат минуя Apple, майнинг на iPhone
- Решение: "Купить RZM" → ведёт на сайт (вне приложения)

"Без цензуры" — ⚠️ риск:
- В приложении для магазинов: базовый фильтр контента
- На сайте (PWA): без ограничений
- Фильтр можно отключить в настройках на сайте, но не в приложении

## 25.3 Если магазины не пропустят

Android (легко):
- APK с сайта (razum.apk) — установка напрямую
- RuStore (российский, лояльный к крипто)
- APKPure, F-Droid, Huawei AppGallery

iOS (сложнее):
- PWA — сайт как приложение, "Добавить на главный экран"
- TestFlight — до 10 000 тестеров без App Store
- AltStore / Sideloading (iOS 17.4+ в ЕС)

## 25.4 Стратегия публикации

1. App Store + Google Play: "чистая" версия (AI-чат, мессенджер, подписки, базовый фильтр, без крипто-кошелька внутри)
2. Сайт (PWA): полная версия (без цензуры, кошелёк, вывод RZM)
3. Android APK + RuStore: полный функционал без ограничений
4. Подписка: через Apple In-App Purchase (30% Apple) + через сайт (0% Apple)
